# 全自動勤怠管理くん — 打刻を、もう意識しない。

WorkOn for Desktop(β版)を参考にした、AIが勤務時間を自動推定する打刻レス勤怠デスクトップアプリ(Electron / Windows・macOS対応)。

## 機能

**従業員側**
- 打刻レス自動記録: `powerMonitor` でPCの実稼働(操作/アイドル/スリープ/画面ロック)を検知し、始業・休憩・終業を自動推定
- 信頼度判定: 安定 / 微妙 / 不安定 の3段階
- マイルール: 勤怠を修正するとAIが差分からルール候補を提案(HITL)。「マイルールに追加 / 今回だけ」を選択でき、以降の推定に自動適用
- カレンダー連携: .ics をインポートすると、会議中の無操作を稼働扱い、「移動」「外出」予定を対象外として提案
- 提出モード4段階: オート(全自動)/ ほどほど(不安定な日だけ手動)/ きっちり(安定した日のみ自動)/ マニュアル
- タイムライン表示、検知イベントログ、履歴からの修正・提出
- トレイ常駐(ウィンドウを閉じても記録継続)、ログイン時自動起動

**案件トラッキング(v0.2.0・オプトイン)**
- 「誰が・何の案件を・どれだけ」を自動計測。判定優先順: カレンダー予定 → 案件コード([A123] / A123_)→ キーワード(顧客名・システム名)
- 未分類の時間はHITLで割り当て。割り当て時に検出語句をキーワードとして学習し、使うほど自動判定率が向上
- 運用ルールは実質2つ: カレンダー予定名に [案件コード] を付ける/新規フォルダ・主要ファイルの先頭に「コード_」を付ける
- プライバシー: ウィンドウタイトルはメモリ上で判定に使用後即破棄。保存されるのは「案件×分数」と未分類ブロックの候補語句のみ
- macOSはタイトル取得に「画面収録」権限が必要(初回に許可ダイアログ)

**動向学習+チーム同期(v0.3.0)**
- 個人の動向学習: 割り当て実績から「語句→案件」(ナイーブベイズ)と「曜日×時間帯→案件」を学習。会議直後は同案件の続きになりやすい「余韻」も加味。確度85%以上は自動計上、50%以上は「AI候補」として提示
- チーム同期(Firebase/Firestore): 案件マスター(キーワードはチームでユニオン)・学習辞書・勤怠/工数サマリーを共有。チームメイトの学習が自分の推定精度にも効く(重み0.4で合成)。管理者ビューはデモではなく実データになり、承認/差し戻しも同期
- 共有されるのは集計値と語句統計のみ。タイトル原文・生ログは送信されない
- セットアップ: [console.firebase.google.com](https://console.firebase.google.com) → プロジェクト作成 → Firestore Databaseを「テストモード」で作成 → 設定画面にProject ID / Web API Key / チームID(全員共通の任意文字列)を入力
- 注意: テストモードのルールは30日で失効します。継続利用時はFirestoreルールの設定(またはテストモード期限の延長)が必要です

**ダッシュボード・通知・収益性(v0.5.0)**
- ダッシュボード: 今日/今週の実働、直近14日グラフ、今月の案件別工数、予算消化バー、粗利テーブル
- デスクトップ通知: 勤怠確定・未提出リマインド(6時間ごと)・承認/差し戻し・予算アラート(設定でオフ可)
- 予算工数アラート: 案件の予算工数の消化80%と100%で通知+リストにバッジ表示
- 収益性: 見積金額 −(実績工数 × 原価単価)で案件ごとの粗利・粗利率を自動計算
- Firestore本番ルール対応: 同梱の `firestore.rules` をコンソールに貼り付け+Authenticationで「匿名」を有効化すると、テストモードの30日制限なしで安全に運用できます(アプリは自動で匿名認証を使用)

**管理者側**
- メンバー勤怠一覧(日付切替)、承認 / 差し戻し
- 提出状況KPI、乖離アラート(提出とPCログの差30分超)
- 人×案件の工数マトリクス(直近30日)とCSV出力
- ※ 本人以外のメンバーはデモデータ(設定から再生成可)

**プライバシー設計**
- 生の稼働サンプルはメモリ上のみで推論に利用し、即時破棄
- アプリ名・ウィンドウタイトル・入力内容は一切収集しない
- 永続化されるのは「何時から何時まで働いたか」の区間と推定結果のみ

## 実行方法

```bash
cd dakoku-less
npm install
npm start        # 起動
npm test         # 推定エンジンのユニットテスト
npm run dist:mac # macOS向けビルド(.dmg)
npm run dist:win # Windows向けビルド(.exe)
```

Node.js 18+ が必要です。データは Electron の userData ディレクトリに `dakoku-less-data.json` として保存されます。

## GitHubで配布(無料)

`.github/workflows/build.yml` により、`v*` タグを push すると GitHub Actions が
Windows(.exe)/macOS(.dmg) のインストーラーを自動ビルドし、Releases に添付します。

```bash
cd dakoku-less
git init && git add -A && git commit -m "initial commit"
git remote add origin https://github.com/<あなたのユーザー名>/dakoku-less.git
git push -u origin main

git tag v0.1.0
git push origin v0.1.0   # → Actionsがビルドし、Releasesページに配布物が並ぶ
```

※ 無料運用のためコード署名なしでビルドします。初回起動時に
macOSでは「右クリック→開く」、Windowsでは SmartScreen の「詳細情報→実行」が必要です。

## トラブルシューティング

**macOS: 「"全自動勤怠管理くん.app"は壊れているため開けません」と表示される**

故障ではなく、署名・公証なしアプリに対するGatekeeperの検疫によるものです。次の手順で解除できます。

1. dmg内の `全自動勤怠管理くん.app` を「アプリケーション」フォルダへドラッグしてコピー
2. ディスクイメージを取り出す
3. ターミナルで `xattr -cr /Applications/全自動勤怠管理くん.app` を実行
4. 以後は通常どおりダブルクリックで起動できます

**Windows: 「WindowsによってPCが保護されました」(SmartScreen)と表示される**

「詳細情報」→「実行」をクリックしてください。いずれも根本解決にはコード署名(Apple Developer Program / コードサイニング証明書)が必要です。

## 構成

```
main.js              メインプロセス(稼働検知・トレイ・自動提出・IPC)
preload.js           contextBridge(安全なIPC公開)
src/engine.js        勤怠推定エンジン(区間結合・空白分類・ルール適用・信頼度・ICS)
src/store.js         JSON永続化
src/demo.js          管理者ビュー用デモチーム生成
renderer/            UI(今日の勤務・履歴・マイルール・設定・管理者ビュー)
test/engine.test.js  エンジンのユニットテスト(10件)
```

## 推定ロジック概要

1. 15秒ごとにシステムのアイドル秒数をサンプリング(閾値は設定可、既定90秒)
2. 稼働区間を結合(3分以内の空白は連続とみなす)
3. 空白を分類: マイルール → カレンダー予定 → ヒューリスティック(15分以上は休憩、11〜14時は昼休憩と推定)の優先順
4. 始業=最初の稼働、終業=最後の稼働。微妙な空白の数などから信頼度を判定
5. 日付切替(既定 深夜4時)で前日分を確定し、提出モードに応じて自動提出
